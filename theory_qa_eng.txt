1. software testing is a process of analyzing software and accompanying documetetinon for indetify defects and improve the quality of the product. 

1.1 testing is a process consisting of all lifecycle activites, both static and dynamic, planning, preparation and evaluation of software products

1.2 debugging is a process of finding, analyzing and removing the causes of failures in software

1.3 exhaustive testing is a test approuch in which the test suite comprises all combinations of input values and preconditions

1.4 pasitive tesing is a tesing aimed at showing software works. also know as "test to pass"

1.5 negative testing is a testing aimed at showing software does not work. also know as "test to fail"

2. software lifecule is the period of time that begins when a software product is coneived and ends when the software is no longer available for use. the software lifecycle typical includes a concept phase, requrements phase, design phase, implemantation phase, test phase, instalation and checkout phase, operation and maintenance phase, and sometimes, retirement phase. 

3. quality assurance is a part of quality management focused on providing confidence that quality requrements will be fulfilled
