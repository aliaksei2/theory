2. что такое API
- application programming interface. API находится между клиентом и сервером. он помогает клиенту правильно обращаться к серверу. оно позволяет связывать между собой разные по структуре приложения.
 
3. что такое rest API
- REST API это способ взаимодействия сайтов и веб приложений с сервером. его также называют RESTful. основывается на протоколе http. REST API применяется везде где необходимо предоставлять данные с сервера пользователю. REST API взаимодействует при помощи http методов (GET, POSt, PUT, DELETE)  
- RESTfull это веб сервис отвечающий требованиям REST

4. требования к rest API
- отделение клиента от сервера ( код запросов остается на стороне клиента, а код для доступа к данным остается на стороне сервера ) 
- отсуствие записи состояния клиента на сервере ( сервер не должен хранить информацию о состоянии клиента. запрос клиента должен содержать только ту информацию, которая нужна для получения данных )  
- кэшируемость (в ответе должно указыватся, является ли ответ кэшируемый. если да, то при последующем запросе, данные будут взяты из кэша, а не из сервера) 
- единый интерфес (предлагет наличие единого способа взаимодействия с сервером вне зависимости от типа устройства или приложения. оно включает в себя : каждый ресурс в REST имеет идентификатор - URL. манипулирование ресурсами через представления. самодокументируемые сообщения. гипермедия как двигатель состояния приложения.)
- многоуровневая система. компоненты могут видеть только те компоненты котоыре расположены на ближайших уровнях и с которыми они взаимодействуют.
- код по требованию. позволяет расширять функциоальность клиеньа за счет загрузки кода с сервера в виде скриптов или апплетов.
 
5. что такое rest and soap
- REST - это архитектурный стиль где REST предоставляет набор требований, которым нужно следовать, чтобы предоставить веб-службу RESTful. REST поддерживает несколько форматов файлов XML, JSON, TXT, CSV, HTML.
- REST это архитектруный стиль в котором описывается структура передачи информации. 
- SOAP - является протоколом обмена сообщниями, по которому веб-сервисы взаимодействуют друг с другом или с клиентом. для общения используется фалы в формате XML иначе сервер вернет ошибку. SOAP более безопасный чем REST. 
- отличия между SOAP and REST:
 rest использует разные форматы файла. soap только xml
 rest работает только с протоколами http/https. soap работает с разными протоколами
 rest может быть закэширован. soap нет
 rest это архитектурный стильно в котором нет огромного количества правил. soap это протокол который сильно ограничен теми правилами которые ему предъявляется
в случае rest это простота, скорость, расширяемость, нет привязки к правилам
в случае soap больше безопасности, и проще для тестирования, разработка дольше и энергозатратна

6. XML, JSON, HTMAL, CSS, JS file
- XML это язые разметки использумый для создания логической структуры данных, их хранения и передачи в виде, удобный человеку и компютеру. синтексис состоит из тегов. в SAOP API испольузется только XML формат, а вот REST API может использовать не только его.
- JSON текстовый формат обмена данными, основанный на Java Script. данный формат данных используется в REST API. структура его состоит из ключ-значение. ключ может быть только стринг, значение может быть в любом типе.
- HTML (HyperText Murkup Lenguege) это код для структурирования и отображения веб страниц и ее контента. он нужен чтоюы размещать на веб сранице элиметы текста,фото,таблицы,видео. он состоит из тегов. в каждом теге есть имя которое содержится в угольных скобках. теги бывают парные и непарные. непарные теги указывают прчто это файл в формате html. парные теги указывают на то что именно будет находится на странице  
- CSS каскадная таблица стилей. это формальный язые декорирования и описания внешнего вида веб страницы, написанного с использованием языка разметки HTML. отвечает за размер, стиль, цвет и оформление страницы.
- JS file это текстовый файл применяемый в целях обслуживаня кода. выполнят функцию интерактивности веб страниц ( переход на другую страницу при нажати на кнопку, анимации изображение, выплывающие окна и тому подобное.

7. что такое http
- Hyper text transfer protocol предназначен для передачи файлов между клиентом и сервером. по правилам этого протокола, сервер может отвечать только после того как получит запрос от клиента. http это просто свод правил для передачи файлов по интернету. он работает поверх протоколов TCP/IP.

8. из чего состоит запрос http
-http метод (описывает что должно быть сделано с ресурсом) 
-endpoint (содержит URI который указвает где и как найти ресурс в интернете)
-header (содержит в себе информацию относящуюся к клиенту и к серверу. предоставляет API ключ, название или IP адрес комютера на котором установлен сервер, информация о формате ответа)
-body (необходимо для передачи дополнительной информации. условно информация о том что хотим добавить или изменить)

9. из чего состоит ответ http
- стартовая строка ответа hhtp (версия протокола http/1.1, статус код, статус текст)
- header
- body

10. какие методы запросов http бывают и какие безопасны
- GET запрос информации (безопасный)
- HEAD запрашивает информацию как GET, только без тела ответа (безопасный)
- POST отправка данных на сервер (не безопасный)
- PUT обновление ранее отправленных данных (не безопасный)
- PATCH частичное изменение данных (не безопасный)
- TRACE вызов возвращаемого тестового сообщения 
- OPTIONS описание параметров соединения с ресурсом (безопасный) 
- CONNECT установление связи с сервером 
- DELETE удаление данных (не безопасный)

11. какие методы идепотенты и что это такое
- идепотентность это если повторный идентичный запрос сделанный один или несколько раз подряд имеет тот же самый эффект, не изменяющий сервер.
- идепотентные : GET, HEAD, PUT, DELETE.

12. что такое header и что в него входит
- header позволяют клиенту и серверу отправлять дополнительную информацию. в нем содержится не чувствительное к регистру название. они могут содержать описание данных и информацию необходимую для взаимодействия между клиентом и сервером.
   
14. какие есть статус коды
- 1** информационные(informational) 
  100 продолжить(continue) (клиент может продолжать отправлять запросы)
  101 переключение протокола(switching protocol) 
- 2** успешные(successful)
  200 ОК
  201 создано (created)
  202 принято (accepted) (запрос принят но еше не обработан)
- 3** переадресация(redirection)
  300 множественный выбор (multiply choice)
  301 перемещен на постоянной основе (moved permanently) (URI был изменен)
  302 перемещен временно (found(moved temporarily))
  303 see other
- 4** ошибка на стороне клиента(client error responses)
  400 плохой запрос (bad request)
  401 не авторизован (unautorization)
  403 запрещено (forbidden) (нет права доступа)
  404 not found
  405 метод не разрешен (method not allowed)
  418 i'm teapot
- 5** ошибка на стороне сервера(server error responses)
  500 внутренняя ошибка сервера (internal server error)
  503 сервис не доступен (servis unavailable)
  504 timeout 
  505 версия http не поддерживается (HTTP version not supported)

15. что такое интерфейс

- интерфейс это инструмент взаимодействия с помощью которого одна система взаимодействует с другой.

16. что такое IP
- это индетификатор позволяющий передавать информацию между устройствами в сети. в него входит информация о месторосположении устройства и обеспечивает его доступность для связи. диапозон IP ареса 0.0.0.0 до 255.255.255.255.
- IP это адрес компьютера в сети.

17. что такое порт
- порт это чисто, указываемое при запросе на соединение с сервером. через него иидет подключение к серверу.
- порт это цифровой индитификатор программы, которая используется для создания определенного интернет соединения привязанног к IP адресу. на сервер находится несколько приложений, и номер порта помогает определить к какому приложению подключится.
- порт для http 80
- порт для https 443

18. что такое хост
- любое устройство, предоставляющее сервисы форматы клиент-сервер в режиме сервера по каким-либо интерфейсам и уникально определенное на этих интерфейсах. в широком смысле хостом могут считать любой компьютер подключенных к локальной или интернет сети.
- хост как уникальный посетитель веб ресурса это любой интернет пользователь попавший на сайт.
- хост это любое устройство, которое имеет подключение к локальной или к интернету и при это является составной частью этой сети. устройство может быть как физическим (телефон, компьютер, маршрутизатор), так и виртуальным.

19. что такое URL URI URN
- URI это строка символов которая используется для идетификации какого либо ресурса по его адрес или по имени, либо по тому и по тому вместе (http://thinkzarahatke.com/author/amty.html#posts)
- URL это строка символов которая используется для индетификации какого либо ресурса, но только по его адресу, по его местоположению (http://thinkzarahatke.com/author/amty.html)
- URN это строка символов которая используется для индетификации какого либо ресурса, но только по его имени (thinkzarahatke.com/author/amty.html#posts)

21. толстый/тонкий холодный/горячий клиент
- толстый клиент это клиент который берет на себя все вычеслительные функции
- тонкий клиент это клиент который имеет возможность только получать готовый резуотат от сервера. все вычеслительные функции происходят на сервере
- холодный клиент это клиент про которого сервер ничего не знает, на стороне клиента не хранится никакой информации про него (обычно это ползователь который первый раз зашел на сайт либо можно включить режим инкогнита)
- горячий клиент это клиент про которого есть информация о его предпочнениях, настройках и так далее.

22. что такое КЭШ КУКИ ЛОКАЛ/СЕССИОН СТОРЕДЖ
- coocie это файлы создаваемые веб сайтами котоыре посещает юзер. в ней хранится информация об вашей атентификации, о предпочтениях и настройках юзера.
- кэш память содержит определенные файлы из сайта (изображения или видео). это надо чтоб в следующий раз сайт прогружался быстрее.
- local storage это веб хранилище где хранится информация ключ значение. информация обновляется после каждого обновления сраницы и даже после перезапуска браузера.
- session storage это веб хранилище которое сохраняет информацию в формате ключ значение, и обновляет информацию каждый раз при обновлении страницы. в случае закрытия вкладки браузера или полное закрытие браузера, данные блокируются.

23. что такое сессия
- сессия это пользовательский сеанс, время посещения сайта. начинается сессия когда юзер заходит на сайт, а заканчивается через пол часа после того как юзер поеинет сайт (если он не вернется обратно)

24. что такое аутентификация, индетификация, валидация, верификация, авторизация
- аутентификация это проверка подлинности заявленного пользователя (првоерка пароля) 
- индетификация это проверка пользователя по личному индитификатору (проверка что такой пользователь есть)
- авторизация это предоставление пользователю определенных прав на сайте.
- верификация это предоставление объективных свидетельств того что данный объект полностью удовлетворяет установленным требованиям (проверка что документ написан корректно и адекватно) 
- валидация это верификация при которой установленные требования связаны с предпологаемым использованием. (проверка что функция отработала так как написано в документации) 

25. как происходит авторизация на сервере
- пользователь вводит в браузер свое имя и пароль, после отправялет на сервер. тот проверяет пользователя, аутентифицирует его и шлет обратно уникальный токен (сохранив его в памяти или базе данных)

35. что такое АЯКС
- AJAX <(Asynhronous JS and XML) (технология обращения к серверу без перезагрузки страницы)  подход к построению интерактивных пользовательских интерфейсов веб-приложений, заключающийся в фоновом обмене данными браузера и веб-сервера. объект может как отправлять так и получать информацию в различных форматах, XML, HTML, TXT. самое важное в нем то, что он работает асинхронно. с помощью этой технологии можно осуществлять взаимодействие с сервером без необходимости перезагрузки страницы. это позволяет обновлять содержимое страницы частично , в зависимости от действий пользователя.

36. SSL TLS
- ssl and tls это протоколы шифрования.
- ssl это криптографический протокол, написанный на языке С. который обеспечивает безопасность связи в компьютерной сети. он защищает целостность и конфиденциоальность данных с помощью шифрования.
- tls это стандарт для безопасности связи через интернет. он позволяет приложению клиент-сервер взаимодействовать по сети таким образом чтобы предотвратить подслушивание и фальсификацию информации. между сайтом и пользователем заключается сертификат ssl/tls которые и шифруют данные.

37. уровни OSI
-модель OSI (open system interconnection)  характеризует взаимодействие сетевого оборудования. простоыми словами описывает как работают сетевые устройства. она необходима для понимания как работают компьютерные сети. другими словами это процесс передачи данных который разделен на 7 уровней: 
1 уровень. физический, на нем происходит момент передачи физических сигналов,токов,импульсов. проще говоря это уровень проводов(блютуз,вайфай) на этом уровне рабоатют концетраторы и репиторы.
2 уровень. канальный. на этом уровне проверяется и исправляются ошибки передачи(информация об отправителе и получателе).
3 уровень. сетевой. на этом уровне происходит маршрутизация трафика. на этом уровне живет маршрутизатор
4 уровень. транспортный обеспечивает передачу данных по сети. он предназначен для надежной передачи информации от отправителя к получателю. тут обитают протоколы TCP and UDP. трафик чувствительный к потерям - TCP. если немного потеряем данных и это не стращно - UDP.
5 уровень. сеансовый. управляет соединением сеансами.
6 уровень. представления. происходит кодирование или сжатие файлов. преобразование.
7 уровень. приложения. этот уровень нужен чтобы приложения имели доступ к сетевым службам. доступ к файлам, пересылкам почты и запросам к базам данным.
 
38. TCP, TCP IP, уровни
- TCP/IP в настоящий момент самый популярный набор протоколов TCP/IP, который задает стандарты связи между компьютерами и содержит подробные соглашения о маршрутизации и межсетевом взаимодействии. основвная функция данного набора это : электронная почта, передача файлов между компьютерами и удаленным выходом в сисетму.
- TCP выполняет проверку доставку всех файлов адресату и проверяет не было ли ошибки при передаче информации. TCP может повторно запросить данные если они были потеряны и регулировать загрузку сети. 
уровни : 1) канальный уровень. 2) межстековой уровень. 3) транспортный. 4) прикладной
- TCP/IP набор сетевых протоколов. передаваемый пакет проходит несколько уровней (как на почте: сначала пишем письмо, потом помещаем в конверт с адресом, затем на почте на нем ставится штамп и так далее.) 
- IP протокол это протокол сетевого уровня. задача этого уровня доставка IP пакетов от отправителя к получателю. кроме собственных данных, пакеты этого уровня имеют IP адрес отправителя и получателя. номера портов на сетевом уровне не используются, а также на этом уровне неизвестно, был ли пакет доставлен.
- TCP и UDP протоколы транспортного уровня. он находится над сетевым. на не к пакету добавляется порты отправителя и получателя. 
- TCP протокол с установлением соединения с гарантированной доставкой пакетов. если пакет не дошел, то он посылается повторно.
- UDP протокол без установления соединения и с негарантированной доставкой пакетов.
- над транспортным уровнем находмтся прикладной уровень, на котором работают такие протоколы как http, ftp и другие. например http и ftp используют надежный протокол TCP, DNS сервер работает через UDP.
 
39. что такое FTP
- протокол передачи данных. это набор правил которые устройства в сети TCP/IP (интернет) исползуют для передачи файлов. когда пользуемся интернетом, мы используем много разных протоколов. например для просмотра веб страницы http.  для отправки и получения мгновенных сообщениий XMPP. для перемещеня файлов FTP.

41. что такое веб сервис/сервер, разница
- веб сервер это машина с определенными характеристиками и приложением, которая хранит в себе информацию и по запросу от клиента, он отправляет ему запрашиваемую информацию
- веб сервисы это набор небольших сервером, которые выдают информацию по запросу от веб сервера . только веб сервер может общаться с веб сервисом. они нужны для безопасности и для снятия нагрузки с веб сервера.

42. man in the middle (MITM) 
- это кибер атака, при котором злоумышленники перехватывают разговор или передачу данных путем подслушивания, либо притворяясь его легальным участником. суть этой атаки кража конфиденциальных данных (банковские карты, номера, перехват платежных средств). по сколько это происходит в реальном времени, то такое отследить почти нереально. самое распространенное это делать ложную вайфай точку, через которую смогут украсть данные тех, кто подключился. самый лучший способ обезпаситьсебя, это заходить на сайты только httpS и использовать VPN

43. web services description language и Web Application Description Language  что это
- WSDL это язык описания веб сервисов. это стандартный формат для описания веб службы. основан на XML протоколе для обмена информацией. он содержит информацию о конечных точках или интерфесах для подключения к внешнему миру. как правило для SOAP.
- простыми словами, он используется для простого описания предоставляемых услуг системой SOAP
- данный файл описывает структуру веб сервиса, как он будет взаимодействовать с запросами. он обязателен для SOAP, без него не получится использовать данный протокол
WSDL можно разделить на логические части: определение типов данных (type), елементы данных (message), абстрактные операции (portType), связывание сервисов (binding), конкретная точка вызова сервиса (service)
<message name="getTermRequest">
   <part name="term" type="xs:string"/>
</message>

<message name="getTermResponse">
   <part name="value" type="xs:string"/>
</message>

<portType name="glossaryTerms">
  <operation name="getTerm">
      <input message="getTermRequest"/>
      <output message="getTermResponse"/>
  </operation>
</portType>

- WADL это язык описания веб приложения. это описание XML в машинном виде веб приложений на основе http. моделирует ресурсы, предоставленные службы и взаимосвязи между ними. оно служит для упрощения повторного использования веб служб, основанных на существуещей архитектуре http. помогает повторно использовать приложения за пределами простого использования в веб браузере. как правило для веб сервисов REST.(машино читаемое XML описание для веб приложений http).
