1. adb это утилита для командной строки для покдлючения и управления телефонов через командную строку на компьютере

2. команды:
- ./adb devices проверка подключенных деввйсов
- ./ adb reboot перезагрузка телефона
- ./ adb install -f app/debug.apk установка apk файла
- ./ adb uninstall com.adroid.todolist (указывается название приложения как в логах) удаление приложения
- ./adb logcat показать логи
- ./adb -s emulator-5554(имя девайса) logcat показать логи
- ./adb logcat | grep -w "com.qrolic.reminderapp > logs.log находит логи по определенному слову и копирует их в новый файл

3. размножить приложение в симуляторе на xcode
-зайти в раздел targets->FlappyBird
- Build settings
- Packaging -> Product Bundle Identifier -> Release

4. посмотреть логи в xcode
- lounch simulator
- debug -> open system log

5. посмотреть логи на айфоне через xcode
- подключаем айфон к ноуту
- window -> devices and simulators
- на телефоне переходим в настройки -> Privacy & Security -> Developer mode -> on
- в xcode нажимаем open console

6. app activity
- activity это один "экран" приложения. его можно сравнить со страницей в вебу. например в приложении "email" могут быть такие activity как "inbox activity", "email activity", "settings activity" и другие.
-activity одного приложения могут запускать друг из друга, так и независимо друг от друга. Если в вебе на другую страницу обращаются по URL, то в android activity запускаются через intent.
- intent это сообщение которое указывает системе что надо сделать (например открыть данные URL, или написать письмо на данный адрес и так далее).

7. app store review
 1) безопасность:
- недопускается оскорбительный контент
- пользовательский контент (быстро реагировать на жалобы клиентов, сохранять авторское право, иметь контакты чтоб пользователи могли связаться)
- детский контент (недопустимо собирать какие либо данные о детях, их устрйоствах и личную информацию. нельзя использовать ссылки и платежные системы)
- физический вред (недопускаются медицинские приложения которые не могут гарантировать точность и долголетие обслуживания приложения, всегда напоминать о консультации с врачом перед принятием решения о лечении. недопускается поощерение принятия табака, наркотиков, алкоголя и так далее)
- связь с разработчиками (должна быть актуальная контактная информация по которой клиент может связаться с разработчиками)
- безопасность данных (должны быть выполнены все требования по лицензии безопасности от APPLE)
 
 2) производительность
- полнота приложения (приложение должно быть полность готово и протестировано, все функции и сервисы включены, все лишнее удалено)
- бета тестирование (все любые тестовые билды не должны быть в app store)
- точные метаданные (должно быть четкое и полное описание о приложении, чтоб клиент понимал что именно он скачивает или покупает. названия приложения должно содержать не больше 30 символов
