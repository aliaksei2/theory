1. adb это утилита для командной строки для покдлючения и управления телефонов через командную строку на компьютере

2. команды:
- ./adb devices проверка подключенных деввйсов
- ./ adb reboot перезагрузка телефона
- ./ adb install -f app/debug.apk установка apk файла
- ./ adb uninstall com.adroid.todolist (указывается название приложения как в логах) удаление приложения
- ./adb logcat показать логи
- ./adb -s emulator-5554(имя девайса) logcat показать логи
- ./adb logcat | grep -w "com.qrolic.reminderapp > logs.log находит логи по определенному слову и копирует их в новый файл

3. размножить приложение в симуляторе на xcode
-зайти в раздел targets->FlappyBird
- Build settings
- Packaging -> Product Bundle Identifier -> Release

4. посмотреть логи в xcode
- lounch simulator
- debug -> open system log

5. посмотреть логи на айфоне через xcode
- подключаем айфон к ноуту
- window -> devices and simulators
- на телефоне переходим в настройки -> Privacy & Security -> Developer mode -> on
- в xcode нажимаем open console

6. app activity
- activity это один "экран" приложения. его можно сравнить со страницей в вебу. например в приложении "email" могут быть такие activity как "inbox activity", "email activity", "settings activity" и другие.
-activity одного приложения могут запускать друг из друга, так и независимо друг от друга. Если в вебе на другую страницу обращаются по URL, то в android activity запускаются через intent.
- intent это сообщение которое указывает системе что надо сделать (например открыть данные URL, или написать письмо на данный адрес и так далее).

7. app store review
 1) безопасность:
- недопускается оскорбительный контент
- пользовательский контент (быстро реагировать на жалобы клиентов, сохранять авторское право, иметь контакты чтоб пользователи могли связаться)
- детский контент (недопустимо собирать какие либо данные о детях, их устрйоствах и личную информацию. нельзя использовать ссылки и платежные системы)
- физический вред (недопускаются медицинские приложения которые не могут гарантировать точность и долголетие обслуживания приложения, всегда напоминать о консультации с врачом перед принятием решения о лечении. недопускается поощерение принятия табака, наркотиков, алкоголя и так далее)
- связь с разработчиками (должна быть актуальная контактная информация по которой клиент может связаться с разработчиками)
- безопасность данных (должны быть выполнены все требования по лицензии безопасности от APPLE)
 
 2) производительность
- полнота приложения (приложение должно быть полность готово и протестировано, все функции и сервисы включены, все лишнее удалено)
- бета тестирование (все любые тестовые билды не должны быть в app store)
- точные метаданные (должно быть четкое и полное описание о приложении, чтоб клиент понимал что именно он скачивает или покупает. названия приложения должно содержать не больше 30 символов)
- совместимость оборудования (по возоможности прилодение должно рабоать на всех устройствах. не стесняться добавлять разные контроллеры к играм. если это приложение на компьютер то не должно требовать высокий доступ, изменять код и лазить по файлам. рабоать на операционной системе что установлено на компьютере и иметь все язык в одной папке)
- требования к ПО (использовать только общедоступное API, использовать фреймворк и api только для основной задачи приложения. постепенно отказываться от устаревших функций. видео транслируемое больше 10 минут по мобильной сети должно использовать http live streaming и включать базовый поток http live 192 кбит/с. не должно быть представлено приложение с пустыми рекламными банерами. для аутентификации с помощью лица нужно использовать localAuthentificration. клипы в приложении не должны содержать рекламу)

3) бизнес
- если приложение будет стоить не разумно дорого - будет отказ. нужно явно дать понять какая бизнес модель описав это в app review.
- платежи (использовать плетежную систему только внутри приложения. иметь возможность хранить внутреннюю валюту приложения без ограниченря и иметь способонсть восстанавливать ее)
- подписки (приложения могут предлагать автопродление подписки. подписка должна быть доступна на всех устройствах пользователя. пользователь должен иметь доступ к повышению или к понижению подписки. перед оформлением подписки, пользователь должен быть четко проиформирован что он поличит при покупки подписки)

4) дизайн
- подражатели (не стоит копировать дизайн других приложений)
- минимальная функциональность (приложение должно выполнять минимальный функционал полезный для пользователя. приложение не должно быть коллекцией ссылок, рекламой и т.д)
- спам (не создавать прилодения которые уже переполняют app store)
- расширение (расширения должны работать на полседней версии safari, не содержать рекламу, не собирать лишнюю информацию, если это стикеры то не должны оскорблять людей и не нарушать закон какой либо страны)
- сайты и служба apple (приложения могут использвать RSS-каналы. можно использовать apple music  вприложении только поверхносто. нельзя использовать служюы apple в качетсве распространителя спама, фишигов и другую нежелательную иформацию. также нельзя собирать данны пользователей из game centre. запрещено использовать push-уведомления в качестве рекламы акции и тому подобное. запрещается использовать emoji apple на других платформах или встраивать в двоичный файл)
- альтернативные иконки приложения (изменять иконку приложения можно только если пользователь сам это выбрал либо с обновлением приложения, иконка должна поменяться во всей системе(настройках, уведомлениях,рабочем столе))
- вход в систему с помощью apple (если прилодение требует аутентификации или создания учетной записи с помощью google или что то еще, всегда должна быть возможность сделать это через apple аккаунт)

5) юридические требования 
- конфиленциальность (защита личных данных пользователей стоит на первом месте. все приложения должны включать в себя ссылку на свою политику конфиденциальности где будет все четко и понятно описано. если ваше прилодение собирает какие либо данные, то пользователь должен быть предупрежден и он должен дать разраешение на это. сбор информации должен быть строго такой, чтоб покрывало функциональность приложеня и не больше. приложение должно учитывать настройки прав доступа установленные на телефоне и придерживаться их. не собирать информацию о пользователе для передечи их третьим лицам или в личных целях)
- дети (личная информация о детях должена храниться строго по правилам хранения конфеденцилаьной информации детей, по всем международным правилам. родительский контроль и согласие родиетелей это разные понятия. контент для детей должен находится строго в разделе для детей. в приложении не должено быть ничего лишнего что может помешать безопасному детскому использованию)
- службы местоположения (использовать только тогда когда это необходимо и если есть разрешение)
- интелектуальная собственность (убедиться что приложение содержит контент созданный вами или есть права на использования чужого контента. приложение не должно поощерять и помогать скачивать чужой контент. не создавать приложение которое будет похоже на приложение apple)
- VPN приложения (нужно использовать только api nevpnmanager. предупреждать пользователя какие данные будут собираться и для чего. если надо, иметь лицензию для предоставления vpn услуг)
- кодекс поведения разработчика (быть вежливым и уважать клиента. не вступать в споры, не грубить и не преследовать. не используйте личную информацию о клиенте в ответ на его отзыв. вся информация о разработчике и владельце приложения должна быть правдивой актуальной и свежей. машеничество строго запрещено. убедитесь что ваше приложение имеет выское качество)



после отправки приложение на app review, команда app store review приступит к обрабтке в ближайшее время и постарается сделать в кратчайшие сроки. если приложение многоступенчатое и сложное, то расммотрение его займет какое то время. так же после каждой ступени проверки, будет обновляться статус приложения и разработчик всегда будет знать в каком состоянии находится приложение. если дата релиза назначена на будущее, то приложение не появится до данной даты даже если оно пройдет проверку до даты релиза. если не согласен с решением команды app review можно подать аппиляцию где будет рассмотрено ваша жалоба.
